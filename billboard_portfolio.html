Project overview

What this project does

Popular music is measurably different today from what it was in 1992. This project quantifies exactly how — by building an automated pipeline that collects three decades of chart data, enriches it with Spotify's acoustic measurements, and surfaces trends, clusters, and patterns that are invisible to casual listening.

Era 1

The 90s

1992 – 1999 · 800 tracks

New Jack Swing, grunge, and R&B domination. High acoustic variety — Boyz II Men and Sir Mix-a-Lot occupy the same chart. Genre lines are sharp; crossover is unusual. Valence tends high; tempo varies widely.

r&b dominance high valence acoustic variety

Era 2

The 2000s

2000 – 2009 · 1,000 tracks

Pop–R&B fusion peaks. Dance pop, teen pop, and hip-hop converge. Electronic production increases danceability and energy. Genre tags multiply; artists blend across clusters. Taylor Swift, Rihanna, and Pink enter the dataset.

dance pop rise hip-hop x pop genre blending

Era 3

The 2010s–20s

2010 – 2022 · 1,200 tracks

Streaming-era compression: louder, shorter, more speech-heavy. Trap and drill elevate speechiness. Electronic production flattens timbre variance. Latin and K-pop infiltrate the Hot 100 in the 2020s.

speechiness spike latin crossover loudness war

Research questions

  • • Has popular music become emotionally more negative over the past 30 years?
  • • Which music genres have risen and fallen in chart dominance during this period?
  • • Do genre clusters share similar acoustic signatures?

Key methodological decision

Running Min-Max normalisation on audio features before trend plotting — rather than Standard (z-score) scaling — preserves the natural 0–1 bounds of Spotify's features and produces directly comparable trend lines. Standard scaling would distort the relative floor and ceiling of each feature.

No labelled training data

Cosine similarity + hierarchical clustering discover genre structure purely from co-occurrence patterns in Spotify's own free-form genre tags. No manually annotated genre taxonomy is required — the clusters emerge from the data.

Methodology

How it was built

Six analytical stages run in sequence — from raw HTML scraping through to a network graph of genre relationships. Each technique was chosen to answer a specific question; the decisions below document what was used and why, including where a simpler approach was deliberately rejected.

01

Stage 1 · Data collection

Wikipedia scraping + Spotify API enrichment

BeautifulSoup scrapes each year's Billboard Year-End Hot 100 Wikipedia page. The wikitable class is parsed row by row to extract rank, title, and artist. Two Spotify API calls are then made per track — one for artist genres, one for the audio features vector — via the Spotipy wrapper.

⚠ Note: 2016 is excluded due to a formatting inconsistency in the Wikipedia source page. Always print the first 3 rows after each year's scrape to confirm the table was parsed correctly — a missing column silently shifts all values.
02

Stage 1 · API enrichment

Spotify audio features

Two calls per track: sp.search(artist) for genre tags, then sp.audio_features([track_id]) for the 12-feature acoustic vector. Both return empty structures on no-match rather than raising, keeping the pipeline from halting mid-loop. Uses SpotifyClientCredentials via the Spotipy wrapper.

✦ Spotify's Audio Features endpoint was restricted to paid/approved tiers in 2024. The pre-collected dataset is included as billboard_tracks.csv for full reproducibility.
03

Stage 2 · Feature engineering

JSON flattening + Min-Max normalisation

The Spotify API returns audio features as a nested dict per row. df['Audio Features'].apply(pd.Series) expands this into individual columns. MinMaxScaler then normalises all numeric features to [0, 1] so trend lines for tempo, energy, valence, danceability, and speechiness sit on the same axis.

✦ Min-Max was chosen over Standard scaling because Spotify features are already designed to be bounded (0–1 natively for most). Standardising would warp the natural floor and ceiling, making "danceability = 1.0" visually equivalent to an outlier rather than the defined maximum.
04

Stage 3 · Temporal analysis

Temporal averaging — full 100 and top-20 sensitivity check

Audio features are averaged per year using groupby('Year').mean() and plotted as overlapping line charts. A second pass filters to the top 20 ranked songs per year to test whether the same trends hold for the chart's dominant tracks, or whether they are diluted by lower-ranked entries.

✦ Running the analysis on both the full 100 and the top 20 is a deliberate sensitivity check — not redundancy. If the top-20 trends diverge from the full-100 trends, the most-played songs are acoustically distinct from the rest of the chart.
05

Stage 4 · Artist analysis

Most-charted artists — feature fingerprinting by artist

The top 5 most frequently appearing artists are identified with value_counts().head(5). Each artist's normalised feature averages are plotted side by side, producing an acoustic "signature" for each — and revealing whether an artist's sound is consistent across years or shifts across the dataset period.

⚠ Note: Artist name matching is exact string comparison against the scraped chart data. Artists listed with featured acts (e.g. "Drake featuring Rihanna") will not match either artist individually — feature credits inflate misses.
✦ Top 5 in the 30-year dataset: Taylor Swift, Mariah Carey, Rihanna, Pink, Kelly Clarkson — a profile that itself tells a story about which artists sustained mainstream presence across decades.
06

Stages 5 & 6 · Unsupervised ML + Visualisation

Genre clustering — cosine similarity → Ward's hierarchical → NetworkX graph

Each Spotify genre tag is binary-encoded as a column — 1 if the track's artist carries that genre, 0 otherwise. cosine_similarity(genre_matrix.T) computes pairwise genre similarity, normalised for frequency. Ward's linkage hierarchical clustering is applied to this similarity matrix, minimising within-cluster variance at each merge. 7 clusters are extracted with fcluster(Z, 7, criterion='maxclust') and labelled via keyword matching (e.g. any genre containing "pop" → Cluster 5). The 7 clusters become nodes in a NetworkX graph; edges are weighted by mean cosine similarity between cluster pairs, with the spring layout (seed=42) placing structurally similar clusters closer together. Edge weight is drawn as line width — a thick edge between Soul & Jazz and Pop indicates heavy tag co-occurrence.

✦ Cosine similarity is preferred over Euclidean distance for sparse binary vectors. Two rare genres that always co-occur together score highly similar even if they appear in only a handful of tracks. Euclidean distance would penalise low-frequency genres unfairly, pulling them toward the origin regardless of their relationships.
⚠ Note: The 7-cluster count is a deliberate interpretive choice, not a data-derived optimum. A dendrogram cut at a different height would produce different groupings — the number was chosen to align with recognisable macro-genre categories.

Full pipeline

# End-to-end pipeline — data collection through network graph
1. Install & import — pip installs; import pandas, requests, BeautifulSoup, spotipy, matplotlib, sklearn, scipy, networkx
2. Scrape Billboard — loop 1992–2022 (skip 2016); Wikipedia wikitable; extract rank / title / artist per row
3. Spotify auth — SpotifyClientCredentials; client_id + client_secret; Spotipy wrapper
4. Enrich: genres — sp.search(artist, type='artist') → artist_info['genres']; empty list on no result
5. Enrich: audio features — sp.search(title+artist, type='track') → track_id → sp.audio_features([id])[0]
6. Build DataFrame — assemble all_years, all_ranks, all_titles, all_artists, all_genres, all_audio_features
7. Flatten features — df['Audio Features'].apply(pd.Series); concat with base columns; drop original dict column
8. Normalise — MinMaxScaler on [valence, energy, danceability, speechiness, tempo]
9. Temporal trends (full) — groupby('Year').mean() → multi-line chart; all 100 songs per year
10. Temporal trends (top 20) — groupby('Year').head(20) → same plot; sensitivity check vs full-100 trends
11. Artist profiling — value_counts top 5; filter df; groupby('Artist').mean(); line chart per feature
12. Peak year per artist — filter per artist; Year.value_counts().idxmax() → top chart year
13. Genre matrix — explode().unique() genre list; binary DataFrame (tracks × genres)
14. Cosine similarity — cosine_similarity(genre_matrix.T) → genres × genres similarity matrix
15. Hierarchical clustering — linkage(similarity_matrix, method='ward') → fcluster(Z, 7, criterion='maxclust')
16. Cluster labelling — keyword matching override: 'pop'→5, 'rap'/'hip hop'→4, 'rock'/'metal'→3, etc.
17. Network graph — nx.Graph(); cluster nodes; weighted edges from similarity_matrix; spring_layout(seed=42)
18. Export — billboard_tracks.csv (full enriched dataset for reproducibility without Spotify API)

Spotify audio features — reference

Feature Range What it measures Used in
valence 0.0 – 1.0 Musical positivity — high = happy/euphoric, low = sad/tense Temporal trends, artist profiles
energy 0.0 – 1.0 Perceptual intensity — fast, loud, noisy production scores high Temporal trends, artist profiles
danceability 0.0 – 1.0 Suitability for dancing based on tempo, rhythm stability, beat strength Temporal trends, artist profiles
speechiness 0.0 – 1.0 Presence of spoken words — >0.66 likely speech-only Temporal trends (rap rise proxy)
tempo BPM Estimated beats per minute Temporal trends (normalised)
acousticness 0.0 – 1.0 Confidence that the track is acoustic (non-electronic) Available; not in primary trend analysis
instrumentalness 0.0 – 1.0 Predicts absence of vocals — >0.5 likely instrumental Available; not in primary trend analysis
loudness dB Overall loudness in decibels, typically –60 to 0 dB Available — relevant to loudness war analysis
key 0 – 11 Estimated key (Pitch Class notation: 0=C, 1=C♯…) Available for harmonic analysis
mode 0 or 1 Modality: 1 = major (brighter), 0 = minor (darker) Available; correlates with valence
duration_ms ms Track length in milliseconds Available — streaming-era shortening hypothesis
time_signature 3 – 7 Estimated time signature (beats per bar) Available; most popular music = 4/4

Key insights & findings

What the data reveals

These are the analytical conclusions the pipeline is designed to surface — each one emerges from a specific stage of the notebook and leads to a concrete follow-up question.

1

Valence has fallen — popular music has measurably got sadder since the 90s

The temporal trend chart shows a clear multi-decade decline in mean valence across the Billboard Hot 100. The 90s produced chart-toppers with high average positivity; by the mid-2010s, sad and melancholic tracks dominate. This tracks the cultural shift documented in academic music research: streaming algorithms amplify slower, more emotionally ambiguous material because it sustains longer listening sessions. → Valence is the single most informative feature for capturing the emotional trajectory of mainstream popular music.

2

Speechiness spikes from 2010 — the chart data captures hip-hop's dominance precisely

Speechiness was the lowest-variance feature throughout the 90s and 2000s, then rises sharply after 2010 and again after 2017. This corresponds exactly to hip-hop becoming the most-streamed genre globally. Speechiness is not a genre label — it is a pure acoustic measurement — yet it reliably tracks the rap share of chart positions without requiring any genre classification. → A single acoustic feature outperforms manual genre annotation as a proxy for hip-hop's chart share.

3

Top-20 trends diverge from full-100 trends — the hits and the chart tell different stories

The sensitivity analysis comparing the top-20 tracks to the full 100 per year reveals that the most dominant chart entries have consistently higher danceability and energy than the rest of the chart. The full-100 valence trend is smoother and less extreme than the top-20 trend — lower-ranked tracks moderate the signal. This means any analysis using only chart position 1–10 is measuring a different musical phenomenon from one using positions 1–100. → Chart rank is a signal, not noise. Scope the analysis to match the hypothesis.

4

Genre clustering reveals that Pop and R&B are structurally the same cluster — not two

Despite being treated as distinct genres in casual discourse, cosine similarity places many contemporary R&B and pop tags into the same hierarchical cluster because the same artists are routinely tagged with both. The network graph makes this visible as a thick edge — heavy co-occurrence. Soul & Jazz, by contrast, sits near but distinct from this cluster. Rap & Hip-Hop forms a tighter, more isolated cluster with fewer bridges to others. → Genre boundaries in pop music are permeability gradients, not hard category lines.

5

The most-charted artists have acoustically distinct signatures — except at crossover

The artist feature fingerprinting shows that Taylor Swift, Rihanna, Pink, and Kelly Clarkson have meaningfully different mean audio profiles — different danceability, energy, and tempo averages — despite all occupying the mainstream pop chart. Mariah Carey sits closest to the R&B cluster centroid. The one convergence point across all five artists is mid-range valence in their most recent entries: mainstream success now requires a narrow emotional range. → An artist's chart longevity is partly acoustic conservatism — sustained presence correlates with staying near the centre of the chart's feature distribution.

Further work

What I'd build next

Three directions that would sharpen and extend these findings — each addresses a known limitation of the current pipeline.

🎧

Extend to streaming data — plays, not chart position

Chart position is a lagging indicator shaped by radio play, sales, and streaming weighed together. A version of this analysis built on raw Spotify stream counts would test whether the acoustic trends found here are driven by what people choose to listen to, or by how the chart algorithm weights different consumption signals. The pipeline is already Spotify-native; adding stream counts is an API call away once the data is accessible.

🌍

Cross-chart comparison — Billboard vs. UK, vs. global charts

The pipeline is built around a URL pattern and a Wikipedia table parser — swapping Billboard for the UK Official Singles Chart or Spotify's global chart requires only a new scraper and the same Spotify enrichment. The research question becomes: do the same valence, energy, and speechiness trends appear in other national charts, or is the Billboard trajectory US-specific?

🤖

Replace rule-based cluster labelling with embedding-based naming

The current genre cluster labels are assigned by keyword matching on genre strings — any genre containing "pop" goes to Cluster 5. This is interpretable but brittle; new genre tags that don't match the keyword list fall to the catch-all "Other" cluster. A sentence embedding of each genre tag, clustered independently, would produce more robust and generalisable labels without manual rule maintenance.

Limitations

Caveats worth knowing

📋 Spotify genres are artist-level, not track-level

Genre tags in this dataset describe the artist's overall profile on Spotify, not the individual track. A country-pop crossover track by an artist tagged "hip hop" will be placed in the hip-hop cluster regardless of the song's actual sound. Track-level genre classification would require a separate audio analysis model — the Spotify genre endpoint does not go below artist granularity.

🗓 2016 is absent — one year creates a gap

The 2016 Wikipedia chart page uses a non-standard table structure that breaks the parser. Excluding it creates a one-year gap in all temporal charts. The trend lines remain valid directionally, but any finding specifically about 2015–2017 should be treated cautiously — the gap lands in a period of significant genre transition (trap's emergence, Drake's Hot 100 dominance).

🔒 Spotify API access is restricted

The Audio Features and Audio Analysis endpoints were restricted to paid/approved developer tiers in 2024. The pipeline's data collection stage cannot be re-run without approved credentials. The included billboard_tracks.csv covers all 3,000 tracks as collected; the analysis and clustering stages are fully reproducible from that file.
