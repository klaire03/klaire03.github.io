Project overview

What this project does

News sentiment is not universal — the same global event can be covered with entirely different emotional tone depending on country, media outlet, and cultural context. This project asks: are two news cultures reporting the world the same way?

Research question

Do geographically and culturally distant news outlets show measurably different sentiment, topic focus, emotional framing, or vocabulary? What topics are covered by both — and what is exclusive to each?

Key methodological decision

Running XLM-RoBERTa on Headline_Original (native Greek) instead of translated English eliminates translation as a confound in the primary sentiment signal. VADER on translated text is kept as a degradation baseline.

No labelled data required

Zero-shot classification, pre-trained transformer models, and unsupervised clustering mean the entire pipeline runs on scraped headlines with no manual annotation. The project is fully replicable on any two news outlets.

Methodology

How it was built

Eight NLP techniques run in sequence from a shared combined_df, each chosen to answer a specific analytical question. Several decisions were made deliberately to maximise cross-linguistic validity.

01

Data collection

🌐 Structured web scraping with polite delays

BeautifulSoup scrapes both outlets using source-specific strategies. To Vima: headlines are stored in WordPress figure img[alt] attributes — a platform quirk where the featured image alt text doubles as the article title. Dates use time.post-date in %d.%m.%Y, %H:%M format. Raw Greek text is stored alongside Google-translated English. Korea Herald: article links match /article/\d+; dates sit as untagged text nodes inside the parent <li>. A 0.5-second polite delay is inserted between page requests. Deduplication runs on URL then on headline text.

⚠ Note: To Vima's WordPress theme stores headlines in figure img[alt], not a heading tag. Parsing the wrong element returns empty strings silently — always print the first 3 rows after scraping to confirm extraction worked.
02

Data cleaning

🧹 Cascading date parser — never drops rows

A three-tier date parser tries source-specific formats first (%d.%m.%Y, %H:%M for Greece; %B %d, %Y for Korea), then ISO 8601, then pandas inference. It never raises — returning NaT on failure. Articles with unparsed dates are kept with NaT rather than dropped. Both datasets are concatenated into a single combined_df with ignore_index=True.

✦ Keeping NaT rows preserves all articles for sentiment, NER, and topic analysis. Silent data loss from a date format edge case would bias temporal analysis without any visible warning.
03

Primary sentiment

🤗 XLM-RoBERTa — multilingual transformer on native text

The Cardiff NLP model (cardiffnlp/twitter-xlm-roberta-base-sentiment) was fine-tuned on 198M multilingual tweets across 100+ languages including Greek. Running it on Headline_Original means Greek articles are scored in Greek — no translation step. The output xlm_compound maps the model's softmax confidence to [−1, 1] by multiplying confidence by the sign of the predicted label. VADER on translated English is kept as a baseline — the agreement rate between methods directly quantifies translation degradation. The .capitalize() call on the model's label string is a necessary defensive step — HuggingFace pipelines can return labels as POSITIVE or Positive depending on model version.

✦ VADER + machine translation introduces noise at every step: changed word choice, lost negation, false cognates. The disagreement rate — typically 30–40% — directly shows how much a VADER-only analysis would have been wrong.
04

Topic modelling

🗂 BERTopic + TF-IDF vocabulary profiling

BERTopic uses sentence embeddings from all-MiniLM-L6-v2 rather than bag-of-words counts. "PM resigns" and "Prime Minister steps down" map to the same semantic neighbourhood regardless of surface wording. Pre-computed embeddings are passed directly to avoid recomputing them for later clustering steps. min_topic_size=5, nr_topics='auto'. A complementary TF-IDF analysis — treating each country's corpus as one document — extracts the most distinctive terms per outlet by IDF penalisation of shared vocabulary. Topic −1 is the BERTopic outlier cluster; always exclude it from distribution charts.

⚠ Note: BERTopic is stochastic — results vary slightly between runs because UMAP and HDBSCAN are both non-deterministic. Pin umap_model and hdbscan_model with random_state=42 for reproducible topics.
05

Topic classification

🏷 Zero-shot NLI — no labelled training data needed

Zero-shot classification frames each headline as an NLI entailment problem: "Does this headline entail the topic 'crime and justice'?" The model (cross-encoder/nli-MiniLM2-L6-H768, ~22M params) was trained on MNLI to answer exactly these questions. Eight thematic categories cover the full editorial spectrum. Batched inference with batch=16 provides 8–10× speedup over one-at-a-time inference on CPU. The original BART-large reference cell is kept in the notebook to document the performance problem explicitly.

✦ The smaller MiniLM model was chosen over BART-large (~400M params) for 8–10× faster CPU inference. The original BART-large reference cell is kept in the notebook so the optimisation decision is visible, not hidden.
⚠ Edge case: when the final batch contains only one item, the HuggingFace pipeline returns a dict instead of a list. The isinstance(results, dict) guard prevents the last batch's label being silently dropped.
06

Named entity recognition

🔍 spaCy batch NER — what names appear in each outlet?

spaCy's en_core_web_sm model runs over translated headlines via nlp.pipe() for batch efficiency — significantly faster than looping nlp() per headline because it amortises model overhead across the batch. Entities are filtered to PERSON, ORG, GPE, and NORP — the four types most analytically relevant to news coverage. Set intersection across both outlets reveals the shared international news agenda — entities mentioned by both media cultures simultaneously.

⚠ Note: NER accuracy on short headline text is lower than on full articles — less context to resolve ambiguous names. Always scan the top entity list for obvious misclassifications before citing in analysis.
07

Emotion & readability

😨 NRC Emotion Lexicon + Flesch-Kincaid + event detection

The NRC lexicon (8 emotions: fear, anger, trust, surprise, sadness, disgust, joy, anticipation) is downloaded directly from GitHub rather than via the fragile nrclex library. Scores are normalised by total emotion-bearing words. Three readability metrics capture headline complexity: FK grade, average word length, and word count. Event detection computes daily sentiment means per country, calculates per-country z-scores (not global — to avoid flagging normal Greek days as spikes if Greek coverage is systematically more negative), and flags spikes at |z| > 1.5. The threshold of 1.5 rather than the conventional 2.0 is deliberate — daily means from a small scrape window have high variance.

⚠ Note: NRC is word-level with no negation awareness — "no joy" scores positive on joy. Flesch-Kincaid was designed for multi-sentence text; treat absolute grade-level numbers as comparative, not literal, on headline-length text. Always check Article_Count on spike days: a spike with count=1 is likely noise.
08

Semantic clustering

🗺 SentenceTransformers → UMAP → HDBSCAN

384-dimensional sentence embeddings (all-MiniLM-L6-v2) are reduced to 2D with UMAP (metric='cosine', n_neighbors=15, random_state=42), then clustered with HDBSCAN (min_cluster_size=6). Clusters where both countries contribute articles are flagged with ★ as shared international stories — the key comparative output. Clustering on 2D UMAP coordinates rather than full 384D embeddings is faster and gives HDBSCAN a cleaner density landscape. A cluster sentiment heatmap shows mean XLM-RoBERTa sentiment per cluster per country, revealing whether shared stories are framed differently across outlets.

✦ UMAP axes have no interpretive meaning — the 2D chart is a visual sanity check, not a measurement. The ★ flag on shared clusters in the composition table is the direct answer to the project's research question.
⚠ Note: Cluster −1 = HDBSCAN noise/outliers. Exclude from cluster analysis but inspect separately — outliers are often the most unusual stories in the dataset.

Key insights & findings

What the data reveals

These analytical conclusions emerge from specific sections of the notebook — each one leads to a concrete follow-up question or methodological implication.

1

📉 Translation degrades sentiment analysis — and we can quantify exactly how much

The agreement rate between XLM-RoBERTa (native Greek) and VADER (translated English) is typically 60–70%, meaning 30–40% of headlines get a different sentiment label depending on whether you translate first. VADER on Greek has no Greek vocabulary — it scores whatever the translation produces. XLM-RoBERTa processes native text directly. The method comparison violin chart makes this visible at a glance: different distribution shapes mean translation introduced noise.

→ The agreement rate is itself a finding: it's the ceiling on accuracy of any English-only sentiment analysis applied to non-English text.

2

🏷 Zero-shot classification works without any labelled training data

The full 8-category topic classification runs with zero manually labelled headlines. The model was trained on NLI entailment, not news topics — but framing "does this headline entail 'crime and justice'?" is enough to get a working classifier. The original BART-large reference cell is kept in the notebook alongside the optimised MiniLM version so the performance problem is documented explicitly, not hidden.

→ This is a portfolio-ready demonstration that modern NLP can solve classification tasks on new domains and new languages with no training data at all.

3

★ Shared clusters reveal what both news cultures were watching simultaneously

HDBSCAN clusters where both Greece and South Korea contribute articles represent stories with genuine international resonance — both media landscapes independently devoted coverage to the same type of event. These shared clusters are the most analytically interesting output: they show the overlap between two very different news cultures. Exclusive clusters show what each outlet was watching that the other wasn't.

→ The ★ flag on shared clusters in the composition table is the direct answer to the project's research question.

4

😨 Emotional framing differs even when topic coverage overlaps

The cluster sentiment heatmap shows mean XLM-RoBERTa sentiment per cluster per country. A shared cluster that is negative in one country and positive in another means both outlets covered the same type of story with opposite emotional tone — different cultural framing of identical news. The NRC radar chart extends this: the shapes of the two countries' emotion profiles reveal systematic differences in fear, anger, trust, and anticipation across the entire headline corpus, not just shared stories.

→ Combined topic classification + sentiment is the analytical contribution that separates this from a simple sentiment dashboard.

5

📈 Event detection needs per-country baselines, not global thresholds

Z-scores for sentiment spike detection are computed per country rather than globally. If Greek coverage is systematically more negative than Korean, using a cross-country baseline would flag normal Greek days as spikes. Per-country z-scores normalise against each outlet's own baseline. The threshold of 1.5 (not the conventional 2.0) is deliberate — daily means of a small number of articles have high variance, and a stricter threshold finds no spikes at all in a short scrape window.

→ Always check Article_Count on spike days: a spike with count=1 is likely noise; a spike with count=8+ is worth investigating.

Further work

What I'd build next

Three directions that would sharpen and extend these findings — each addressing a known limitation of the current pipeline.

🌐

Extend to full article text

The entire pipeline runs on headlines only — typically 8–12 words. Full article text would produce more reliable sentiment, NER, and emotion scores by giving models more context. The main change is reducing max_length for XLM-RoBERTa (which truncates at 128 tokens) and reducing batch_size for SentenceTransformer to avoid OOM. Event detection would become substantially more reliable on full-article sentiment.

📅

Longitudinal scraping over 3–6 months

The time series and event detection sections work correctly but produce very few data points on a short scrape. Weekly resampling with 2–3 weeks of data gives 2–3 line points per country — directionally correct but not analytically meaningful. A scheduled scraper running daily would accumulate enough data for genuine trend analysis, seasonality detection, and credible event attribution.

🔄

Add more country–language pairs

The pipeline is designed to be generalised — the same code runs on any two outlets by swapping the scraping functions and adjusting date format strings. Adding Arabic, Japanese, or French outlets would test whether the XLM-RoBERTa + zero-shot approach holds across more linguistically distant pairs. The research question becomes: is the Greece–Korea divergence typical or exceptional compared to other cross-national comparisons?

Limitations

Caveats worth knowing

📰 Headlines are not articles

All analysis runs on 8–12 word headlines, not full articles. Sentiment, NER accuracy, and emotion scores are all lower-confidence on such short text — headlines compress nuance and can be deliberately ambiguous or clickbait-oriented. The findings are directionally valid but absolute scores should be treated cautiously.

🌍 Single outlet per country

To Vima and Korea Herald each represent one editorial voice within a much larger media landscape. To Vima is a centrist broadsheet; Korea Herald is English-language and internationally oriented. Both may be unrepresentative of their country's full media discourse. A robust cross-national study would aggregate multiple outlets per country.

⏱ Short scrape window

A typical scrape covers 1–2 weeks of recent headlines. The time series, event detection, and day-of-week heatmap sections work correctly but have very few data points. Conclusions about temporal patterns require at least 4–8 weeks of data per analysis window. Re-run the notebook with a wider scrape to make those sections analytically meaningful.
