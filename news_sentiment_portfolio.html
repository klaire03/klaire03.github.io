Project overview

What this project does

News sentiment is not universal — the same global event can be covered with entirely different emotional tone depending on country, media outlet, and cultural context. This project asks: are two news cultures reporting the world the same way?

Source 1 · Greece

To Vima

tovima.gr · Society category · 6 pages scraped

One of Greece's oldest and most respected broadsheet newspapers. Headlines are extracted from WordPress img alt attributes — a platform quirk where the featured image alt text doubles as the article title. Raw Greek text is stored alongside Google-translated English.

Scraping detail: HTML structure uses figure[style] → img[alt] for headlines; time.post-date for dates in %d.%m.%Y, %H:%M format.

koreaherald.com · National section · 10 pages scraped

Korea's largest English-language daily. Article links match /article/\d+; dates sit as untagged text nodes inside the parent <li>. A 0.5-second polite delay is inserted between page requests. Headlines are already in English — no translation step needed.

Research question

Do geographically and culturally distant news outlets show measurably different sentiment, topic focus, emotional framing, or vocabulary? What topics are covered by both — and what is exclusive to each?

Key methodological decision

Running XLM-RoBERTa on Headline_Original (native Greek) instead of translated English eliminates translation as a confound in the primary sentiment signal. VADER on translated text is kept as a degradation baseline.

No labelled data required

Zero-shot classification, pre-trained transformer models, and unsupervised clustering mean the entire pipeline runs on scraped headlines with no manual annotation. The project is fully replicable on any two news outlets.

Methodology

How it was built

Eight NLP techniques run in sequence, each chosen to answer a specific analytical question. Several decisions were made deliberately to maximise cross-linguistic validity.

01

Data collection

Structured web scraping with polite delays

BeautifulSoup scrapes both outlets using source-specific strategies — img alt extraction for To Vima, link pattern matching for Korea Herald. A 0.5s sleep between Korea Herald pages avoids triggering rate limits. A descriptive User-Agent string is used throughout.

⚠ Note: To Vima's WordPress theme stores headlines in figure img[alt], not a heading tag. Parsing the wrong element returns empty strings silently — always print the first 3 rows after scraping to confirm extraction worked.
02

Cleaning

Cascading date parser — never drops rows

A three-tier date parser tries source-specific formats first (%d.%m.%Y, %H:%M for Greece; %B %d, %Y for Korea), then ISO 8601, then pandas inference. It never raises — returning NaT on failure. Articles with unparsed dates are kept with NaT rather than dropped.

✦ Keeping NaT rows preserves all articles for sentiment, NER, and topic analysis. Silent data loss from a date format edge case would bias temporal analysis without any visible warning.
03

Sentiment

XLM-RoBERTa on native text — not on translations

The Cardiff NLP model was fine-tuned on 198M multilingual tweets across 100+ languages including Greek. Running it on Headline_Original means Greek articles are scored in Greek. VADER on translated English is kept as a baseline. The agreement rate between methods quantifies translation degradation.

✦ VADER + machine translation introduces noise at every step: changed word choice, lost negation, false cognates. The disagreement rate — typically 30–40% — directly shows how much a VADER-only analysis would have been wrong.
04

Topic modelling

BERTopic over LDA — semantic, not lexical

BERTopic uses sentence embeddings from all-MiniLM-L6-v2 rather than bag-of-words counts. "PM resigns" and "Prime Minister steps down" map to the same semantic neighbourhood regardless of surface wording. Pre-computed embeddings are passed directly to avoid recomputing them for later clustering steps.

⚠ Note: BERTopic is stochastic — results vary slightly between runs because UMAP and HDBSCAN are both non-deterministic. Pin umap_model and hdbscan_model with random_state=42 for reproducible topics.
05

Classification

Zero-shot NLI — no labelled training data needed

Zero-shot classification frames each headline as an NLI entailment problem: "Does this headline entail the topic 'crime and justice'?" The model was trained on MNLI to answer exactly these questions. Eight thematic categories cover the full editorial spectrum. The smaller MiniLM model (~22M params) was chosen over BART-large (~400M) for 8–10× faster CPU inference.

✦ The original BART-large reference cell is kept in the notebook to document the performance problem explicitly — a design decision that makes the optimisation visible, not hidden.
06

Named entity recognition

spaCy batch NER — what names appear in each outlet?

spaCy's en_core_web_sm model runs over translated headlines via nlp.pipe() for batch efficiency. Entities are filtered to PERSON, ORG, GPE, and NORP — the four types most analytically relevant to news coverage. Set intersection across both outlets reveals the shared international news agenda.

⚠ Note: NER accuracy on short headline text is lower than on full articles — less context to resolve ambiguous names. Always scan the top entity list for obvious misclassifications before citing in analysis.
07

Emotion & readability

NRC Emotion Lexicon + Flesch-Kincaid

The NRC lexicon (8 emotions: fear, anger, trust, surprise, sadness, disgust, joy, anticipation) is downloaded directly from GitHub rather than via the fragile nrclex library. Scores are normalised by total emotion-bearing words. Three readability metrics capture headline complexity: FK grade, average word length, and word count.

⚠ Note: NRC is word-level with no negation awareness — "no joy" scores positive on joy. Flesch-Kincaid was designed for multi-sentence text; treat absolute grade-level numbers as comparative, not literal, on headline-length text.
08

Semantic clustering

SentenceTransformers → UMAP → HDBSCAN

384-dimensional sentence embeddings are reduced to 2D with UMAP (metric='cosine', n_neighbors=15, random_state=42), then clustered with HDBSCAN (min_cluster_size=6). Clusters where both countries contribute articles are flagged as shared international stories — the key comparative output.

✦ Clustering on 2D UMAP coordinates rather than full 384D embeddings is faster and gives HDBSCAN a cleaner density landscape. UMAP axes have no interpretive meaning — the chart is a visual sanity check, not a measurement.

NLP pipeline

Eight techniques, one notebook

Each component is a deliberate choice with a specific trade-off. The full stack runs on scraped headlines with no manually labelled training data.

🤗

Primary sentiment

XLM-RoBERTa — Multilingual Transformer

cardiffnlp/twitter-xlm-roberta-base-sentiment · 198M tweets · 100+ languages

Scores headlines in their native language. Greek articles are scored in Greek — no translation step. The output xlm_compound maps the model's softmax confidence to [−1, 1] by multiplying by the sign of the predicted label: score × LABEL_MAP[label]. This produces a range directly comparable to VADER without any rescaling.

🇬🇷 Greek: Headline_Original 🇰🇷 Korean: Headline_Original (already EN)
✦ The .capitalize() call on the model's label string is a necessary defensive step — HuggingFace pipelines can return labels as POSITIVE or Positive depending on model version. Without it, LABEL_MAP silently returns 0 for every row, making all compound scores zero.
📊

Baseline comparison

VADER

Rule-based · English-only · Social media optimised

Runs on translated English Headline only. Kept as a methodological baseline — not the primary signal. Agreement rate with XLM-RoBERTa directly quantifies how much sentiment information is lost in translation.

⚠ VADER has no knowledge of Greek. When applied to machine-translated Greek, every step introduces error. The disagreement rate (typically 30–40%) is itself a finding.
🗂

Topic modelling

BERTopic

all-MiniLM-L6-v2 embeddings · UMAP + HDBSCAN internal · c-TF-IDF labels

Clusters headlines semantically, not lexically. Auto-labels each topic with its top 3 c-TF-IDF keywords. Pre-computed embeddings are passed directly to avoid recomputing. min_topic_size=5, nr_topics='auto'.

✦ Topic −1 is the outlier cluster. Always exclude it from distribution charts — it is not a real topic.
🏷

Topic classification

Zero-Shot NLI

cross-encoder/nli-MiniLM2-L6-H768 · ~22M params · 8 categories · batch=16

Classifies each headline into one of eight thematic categories with no labelled training data. Batched inference (16 headlines per forward pass) provides 8–10× speedup over one-at-a-time inference on CPU.

⚠ Edge case: when the final batch contains only one item, the HuggingFace pipeline returns a dict instead of a list. The isinstance(results, dict) guard prevents the last batch's label being silently dropped.
🔍

Named entities

spaCy NER

en_core_web_sm · nlp.pipe batch=64 · PERSON / ORG / GPE / NORP

Extracts named entities from translated headlines in batch mode. Set intersection across both outlets surfaces the shared international news agenda — entities mentioned by both media cultures simultaneously.

✦ nlp.pipe() is significantly faster than looping nlp() per headline because it amortises model overhead across the batch.
😨

Emotional framing

NRC Emotion Lexicon

8 emotions · word-level · downloaded directly from GitHub

Scores each headline on fear, anger, trust, surprise, sadness, disgust, joy, and anticipation as a fraction of total emotion-bearing words. Direct GitHub download avoids the fragile nrclex library dependency.

⚠ No negation awareness — "no joy" scores positively on joy. Treat as comparative, not absolute.
📐

Readability

Flesch-Kincaid + Word Stats

textstat · FK grade · avg word length · headline word count

Three metrics that operationalise editorial complexity from different angles: syllable-based grade level, lexical complexity (avg character length per word), and information density (word count).

⚠ FK grade was designed for multi-sentence prose. Use comparatively across countries, not as a literal reading level for sub-10-word headlines.
🗺

Semantic structure

SentenceTransformers → UMAP → HDBSCAN

all-MiniLM-L6-v2 · 384-dim → UMAP cosine 2D → HDBSCAN min_cluster_size=6

Encodes meaning (not just words) into 384-dimensional vectors, projects to 2D with cosine-metric UMAP for visualisation, then clusters with HDBSCAN. Clusters containing articles from both countries are flagged with a ★ as shared international stories — the primary comparative output of this section.

✦ random_state=42 on UMAP ensures the 2D layout is reproducible, but UMAP axes have no interpretive meaning — they are a visual sanity check only.
⚠ Cluster −1 = HDBSCAN noise/outliers. Exclude from cluster analysis but inspect separately — outliers are often the most unusual stories in the dataset.

Key insights & findings

What the data reveals

These are the analytical conclusions the pipeline is designed to surface — each one emerges from a specific section of the notebook and leads to a concrete follow-up question.

1

Translation degrades sentiment analysis — and we can quantify exactly how much

The agreement rate between XLM-RoBERTa (native Greek) and VADER (translated English) is typically 60–70%, meaning 30–40% of headlines get a different sentiment label depending on whether you translate first. VADER on Greek has no Greek vocabulary — it scores whatever the translation produces. XLM-RoBERTa processes native text directly. The method comparison violin chart makes this visible at a glance: different distribution shapes = translation introduced noise. → The agreement rate is itself a finding: it's the ceiling on accuracy of any English-only sentiment analysis applied to non-English text.

2

Zero-shot classification works without any labelled training data

The full 8-category topic classification runs with zero manually labelled headlines. The model was trained on NLI entailment, not news topics — but framing "does this headline entail 'crime and justice'?" is enough to get a working classifier. The original BART-large reference cell is kept in the notebook alongside the optimised MiniLM version so the performance problem is documented explicitly, not hidden. → This is a portfolio-ready demonstration that modern NLP can solve classification tasks on new domains and new languages with no training data at all.

3

Shared clusters reveal what both news cultures were watching simultaneously

HDBSCAN clusters where both Greece and South Korea contribute articles represent stories with genuine international resonance — both media landscapes independently devoted coverage to the same type of event. These shared clusters are the most analytically interesting output: they show the overlap between two very different news cultures. Exclusive clusters show what each outlet was watching that the other wasn't. → The ★ flag on shared clusters in the composition table is the direct answer to the project's research question.

4

Emotional framing differs even when topic coverage overlaps

The cluster sentiment heatmap shows mean XLM-RoBERTa sentiment per cluster per country. A shared cluster that is red in one country and green in another means both outlets were covering the same type of story, but with opposite emotional tone — different cultural framing of identical news. The NRC radar chart extends this: the shapes of the two countries' emotion profiles reveal systematic differences in fear, anger, trust, and anticipation across the entire headline corpus, not just shared stories. → Combined topic classification + sentiment is the analytical contribution that separates this from a simple sentiment dashboard.

5

Event detection needs per-country baselines, not global thresholds

Z-scores for sentiment spike detection are computed per country rather than globally. If Greek coverage is systematically more negative than Korean, using a cross-country baseline would flag normal Greek days as spikes. Per-country z-scores normalise against each outlet's own baseline. The threshold of 1.5 (not the conventional 2.0) is deliberate — daily means of a small number of articles have high variance, and a stricter threshold finds no spikes at all in a short scrape window. → Always check Article_Count on spike days: a spike with count=1 is likely noise; a spike with count=8+ is worth investigating.

Further work

What I'd build next

Three directions that would sharpen and extend these findings — each addressing a known limitation of the current pipeline.

🌐

Extend to full article text

The entire pipeline runs on headlines only — typically 8–12 words. Full article text would produce more reliable sentiment, NER, and emotion scores by giving models more context to work with. The main change is reducing max_length for XLM-RoBERTa (which truncates at 128 tokens) and reducing batch_size for SentenceTransformer to avoid OOM. Event detection would become substantially more reliable on full-article sentiment.

📅

Longitudinal scraping over 3–6 months

The time series and event detection sections work correctly but produce very few data points on a short scrape. Weekly resampling with 2–3 weeks of data gives 2–3 line points per country — directionally correct but not analytically meaningful. A scheduled scraper running daily would accumulate enough data for genuine trend analysis, seasonality detection, and credible event attribution.

🔄

Add more country-language pairs

The pipeline is designed to be generalised — the same code runs on any two outlets by swapping the scraping functions and adjusting date format strings. Adding Arabic, Japanese, or French outlets would test whether the XLM-RoBERTa + zero-shot approach holds across more linguistically distant pairs. The research question becomes: is the Greece–Korea divergence typical or exceptional compared to other cross-national comparisons?

Limitations

Caveats worth knowing

📰 Headlines are not articles

All analysis runs on 8–12 word headlines, not full articles. Sentiment, NER accuracy, and emotion scores are all lower-confidence on such short text — headlines compress nuance and can be deliberately ambiguous or clickbait-oriented. The findings are directionally valid but absolute scores should be treated cautiously.

🌍 Single outlet per country

To Vima and Korea Herald each represent one editorial voice within a much larger media landscape. To Vima is a centrist broadsheet; Korea Herald is English-language and internationally oriented. Both may be unrepresentative of their country's full media discourse. A robust cross-national study would aggregate multiple outlets per country.

⏱ Short scrape window

A typical scrape covers 1–2 weeks of recent headlines. The time series, event detection, and day-of-week heatmap sections work correctly but have very few data points. Conclusions about temporal patterns require at least 4–8 weeks of data per analysis window. Re-run the notebook with a wider scrape to make those sections analytically meaningful.
